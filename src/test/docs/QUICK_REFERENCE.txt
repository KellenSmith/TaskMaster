🛑 COMMON ISSUES & FIXES
═══════════════════════════════════════════════════════════════════════════════

Test not running?
   Fix: pnpm install
   Then: pnpm test -- --watch

"Cannot find module '@/...'"?
   Fix: Check tsconfig.json paths (already configured)
   Use: import { X } from '@/app/...' not '../../../'

Test times out?
   Fix: Check for missing await keywords
   Use: await userEvent.click()
   Use: await waitFor(() => {...})

Flaky test that sometimes fails?
   Fix: Don't rely on timing
   Use: waitFor() instead of setTimeout()
   Use: vi.clearAllMocks() between tests

Test can't find component?
   Fix: Check file extension (.test.tsx for components)
   Use: .test.ts for utilities, .test.tsx for components

🎓 KEY CONCEPTS
═══════════════════════════════════════════════════════════════════════════════

Testing Pyramid:
         /\                    ← Test here last (complexity ⬆)
        /  \  Pages & APIs
       /────\                  ← Complex components
      /      \
     /        \  Components    ← Simple components
    /________\
   /          \
  /  Utilities \               ← Test here first (speed ⬆)
 /____________\                  (ROI ⬆)

File Organization:
   src/app/lib/utils.ts          ← Source file
   src/app/lib/utils.test.ts     ← Test file (same folder!)

Test Pattern (AAA):
   Arrange → Set up data
   Act     → Run code
   Assert  → Check results
